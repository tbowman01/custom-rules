
# Contributing to Trevor's Custom Rules CLI

## Testing & Quality Standards

### Coverage Requirements
- **Overall coverage:** ≥**90%** lines/branches/functions/statements
- **Unit tests:** **100%** coverage for files under `src/core/**` and `src/utils/**`
- **Process:** **TDD required** (tests land with or before feature code)

### Test Structure
```
tests/
  unit/        # fast, isolated (core, utils) - 100% coverage required
  integration/ # adapters, CLI e2e, template generation
  fixtures/    # test data and golden snapshots
```

## Development Workflow (TDD Required)

### 1. Before Starting Any Feature
```bash
# Ensure you have the latest dependencies
npm install

# Start in watch mode for TDD
npm run test:watch
```

### 2. TDD Development Cycle
1. **Write failing tests first**
   - Unit tests in `tests/unit/` for pure logic
   - Integration tests in `tests/integration/` for CLI behavior
   - Make sure tests fail initially (proves they're testing the right thing)

2. **Implement minimal code** to make tests pass
   - Focus on the core logic first (`src/core/`, `src/utils/`)
   - Then wire up CLI commands (`src/cli/`)

3. **Refactor** with tests green
   - Improve code quality while maintaining test coverage
   - Run `npm run coverage` to verify thresholds

4. **Validate before commit**
   ```bash
   npm run precommit  # Runs lint + typecheck + coverage
   ```

### 3. Pre-commit Requirements
Our Husky pre-commit hook enforces:
- **Linting** must pass (`npm run lint`)
- **Type checking** must pass (`npm run typecheck`)  
- **Coverage thresholds** must be met (`npm run coverage`)
- **TDD policy**: Changes to `src/` must include test changes

### 4. Commands for Development

```bash
# Core development commands
npm test           # Run all tests
npm run test:unit      # Run unit tests only (fast feedback)
npm run test:int       # Run integration tests only
npm run test:watch     # Watch mode for TDD
npm run coverage       # Generate coverage report with enforcement

# Quality checks
npm run lint           # ESLint on all TypeScript files
npm run typecheck      # TypeScript strict type checking
npm run precommit      # Full validation (used by pre-commit hook)

# Build and run
npm run build          # Compile TypeScript to dist/
npm start -- <args>    # Run the CLI with arguments
```

## Testing Patterns

### Unit Tests (100% Coverage Required)
- **Location**: `tests/unit/`
- **Target**: `src/core/**` and `src/utils/**` 
- **Rules**: 
  - No file system operations
  - No process spawning
  - No network calls
  - Use dependency injection for external dependencies
  - Mock all side effects

**Example**:
```typescript
// tests/unit/core/config.test.ts
import { describe, it, expect, vi } from 'vitest';
import { ConfigManager } from '../../../src/core/config.js';

describe('ConfigManager', () => {
  it('should load default configuration when no config file exists', () => {
    // Test pure logic without FS operations
  });
});
```

### Integration Tests (≥90% Overall)
- **Location**: `tests/integration/`
- **Target**: CLI commands, file generation, template processing
- **Patterns**:
  - Use `execa` to test CLI in temporary directories
  - Golden snapshots for template outputs
  - Security validation (path traversal, injection)
  - End-to-end command workflows

**Example**:
```typescript
// tests/integration/cli/init.test.ts
import { describe, it, expect } from 'vitest';
import { execa } from 'execa';
import { mkdtemp, readdir } from 'fs/promises';

describe('init command', () => {
  it('should create configuration files', async () => {
    const tmpDir = await mkdtemp('test-');
    await execa('node', ['dist/cli/main.js', 'init'], { cwd: tmpDir });
    // Assert files were created correctly
  });
});
```

## Code Style and Standards

### ESLint Configuration
- **Unit tests**: `no-console` error (must be mocked)
- **All tests**: Vitest globals enabled
- **TypeScript**: Strict mode, no unused variables

### Error Handling
Always use the standard error handling pattern:
```typescript
try {
  await operation();
} catch (error) {
  console.error(chalk.red('Error:'), error.message);
  if (verbose) {
    console.error(chalk.red('Stack trace:'), error.stack);
  }
  process.exit(1);
}
```

### File Generation
- Use `--dry-run` for preview mode
- Support `--force` for overwriting files
- Generate files with `# Generated by Copilot` headers
- Validate against security filters before any file operations

## Adding New Features

### 1. For Core Logic Changes (`src/core/`, `src/utils/`)
1. **Write unit tests first** in `tests/unit/`
2. Ensure **100% coverage** for the new code
3. Tests must be **isolated** (no FS, network, processes)
4. Use dependency injection for testability

### 2. For CLI Commands (`src/cli/`)
1. **Write integration tests first** in `tests/integration/`
2. Test the **complete command workflow**
3. Include **error scenarios** and **edge cases**
4. Test with temporary directories for file operations

### 3. For Security Features
1. **Write security tests** that attempt exploits
2. Test **path traversal** prevention
3. Test **command injection** blocking
4. Validate against `security.neverExecute` patterns

## Pull Request Process

### Before Opening a PR
```bash
# Ensure all checks pass
npm run precommit

# Run the full test suite
npm test

# Verify coverage meets requirements
npm run coverage
```

### PR Requirements
- **All tests pass** in CI
- **Coverage thresholds met** (90% overall, 100% core/utils)
- **TDD policy followed** (src/ changes include test changes)
- **Type checking passes** with TypeScript strict mode
- **Linting passes** with project ESLint rules

### CI Pipeline
Our GitHub Actions workflow will:
1. Install dependencies with npm
2. Run linting (`npm run lint`)
3. Run type checking (`npm run typecheck`)  
4. Run tests with coverage (`npm run coverage`)
5. Enforce coverage thresholds via JSON parsing
6. Upload coverage reports to Codecov

## Debugging

### Coverage Analysis
```bash
# Generate detailed HTML coverage report
npm run coverage -- --reporter=html
# Open coverage/index.html in browser
```

### Test Debugging
```bash
# Run specific test file
npm test tests/unit/core/config.test.ts

# Run tests matching pattern
npm test -- --grep "config loading"

# Debug mode with watch
npm run test:watch -- --reporter=verbose
```

### CLI Debugging
```bash
# Test CLI commands with verbose output
node dist/cli/main.js init --verbose --dry-run --ide all

# Check configuration status
node dist/cli/main.js info

# Validate specific profiles
node dist/cli/main.js profile show development
```
